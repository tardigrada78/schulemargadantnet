Verwendete KI-Modelle

Die Tools auf dieser Website können verschiedene KI-Modelle verwenden. Hier eine Übersicht der verfügbaren Optionen.

OpenAI Cloud

Modelle von OpenAI (z. B. GPT-4o) werden über die OpenAI-API angesprochen. Dazu wird ein API-Schlüssel benötigt, der auf platform.openai.com erstellt werden kann. Die Anfragen werden an OpenAI-Server gesendet; die Daten verlassen das Gerät.

Anthropic Cloud

Modelle von Anthropic (z. B. Claude Sonnet oder Claude Haiku) werden über die Anthropic-API angesprochen. Ein API-Schlüssel ist erforderlich und kann auf console.anthropic.com bezogen werden. Auch hier werden Anfragen an externe Server gesendet.

Ollama via Cloudflare Tunnel Lokal

Mit Ollama können KI-Modelle lokal auf dem eigenen Gerät ausgeführt werden – ohne dass Daten an externe Server gesendet werden. Über einen Cloudflare Tunnel wird das lokal laufende Ollama für die Website erreichbar gemacht.

Anleitung
  1. Cloudflared installieren – Anleitung und Downloads: developers.cloudflare.com → Tunnel Downloads
  2. Ollama starten – Ollama normal starten (es läuft standardmässig auf Port 11434).
  3. Tunnel öffnen – In der Konsole eingeben:
    cloudflared tunnel --url http://localhost:11434
  4. URL kopieren – Die angezeigte URL (erscheint nach der Meldung „Your quick Tunnel has been created!") in die Zwischenablage kopieren und auf der Website eintragen.
  5. Modell wählen – Auf der Website bei «Modell» eines der installierten lokalen Modelle eingeben. Installierte Modelle können mit ollama list aufgelistet werden.

webLLM Lokal (Browser)

webLLM führt KI-Modelle direkt im Browser aus – ohne Server, ohne API-Schlüssel, ohne Internetzugang nach dem ersten Laden. Das Modell wird einmalig in den Browser-Cache heruntergeladen und danach vollständig lokal ausgeführt. Voraussetzung ist ein aktueller Browser mit WebGPU-Unterstützung (z. B. Chrome 113+). Je nach Modellgrösse wird mehr Arbeitsspeicher benötigt.
